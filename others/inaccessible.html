Inaccessible Websites

Web accessibility problems - Examples

Problem with keyboard access

  • MakeMyTrip.com :- The date field cannot be used without the mouse by a person with motor disability.

information conveyed through Images only

  • snapdeal.com :- The payment methods image is not accessible with the keyboard.
  • Delhi Govt. Portal

Dynamic content not made available

  • The Times of India :- The breaking news section is simply not made available to a screen reader. The cross button is useful for a user who is distracted by moving content
  • MoneyControl.com :- The price ticker is not accessible with any assistive technology.

Rapidly blinking and moving content

  • can cause a seizure
  • Haryana transport department site :-
    Rapid blinking can cause seizures.
    The content is not available to a screen reader.
    Headings used instead of a list. Bad use of HTML.

  • Haryana transport department booking page :-
    Uses a table for layout. Bad HTML. Has changing text that is not made available to AT using aria live region or anything.

Incorrect methods to structure content

  • RTI Website :- Uses a table for layout. No headings or landmarks make usability for a screen reader user almost impossible.
  • Good tutorial on correct table use

User focus is taken away

  • Naukri.com

Links without contextual information

  • MTNL customer care page - has links which when viewed as a list of links have no meaning.

Background music

  • A children's site - music starts with no way for the user to easily stop the music. Very hard to use for a screen reader user.

Bad background

  • poor choice of background
